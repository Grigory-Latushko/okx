# buy-the-dip — покупаем просадку

## Стратегия

Мониторим суточное изменение цены инструментов из конфига. При достижении порога падения — открываем LONG с фиксированными TP% и SL%.

## Логика работы

```
Каждый цикл для каждого инструмента:
  1. Если позиция открыта → показываем статус (entry, now, P&L%)
  2. Если позиция закрылась → логируем в CSV (TP/SL)
  3. Если позиции нет → смотрим суточное падение
     └── если падение >= порога → покупаем LONG с TP и SL
```

## Параметры конфига

| Поле | Описание | Пример |
|---|---|---|
| `instruments` | Список инструментов для мониторинга | `["BTC-USDT-SWAP", "ETH-USDT-SWAP"]` |
| `dip_threshold_pct` | Порог суточного падения для входа | `-5.0` (= -5%) |
| `position_size_usd` | Размер позиции в USD | `1` |
| `leverage` | Плечо | `10` |
| `tp_pct` | Take Profit в % от цены входа | `3.0` |
| `sl_pct` | Stop Loss в % от цены входа | `1.5` |
| `dryRun` | Тестовый режим без реальных сделок | `true` |

## Как считается просадка

Используется поле `open24h` из тикера OKX — цена ровно 24 часа назад.

```
dropPct = (текущая_цена - цена_24ч_назад) / цена_24ч_назад × 100
```

Если `dropPct <= dip_threshold_pct` → сигнал на покупку.

## Запуск

```powershell
.\trade.ps1              # dryRun режим
.\trade.ps1 -ForceLive   # боевой режим
.\trade.ps1 -DebugMode   # debug вывод
```

## ⚠️ Важно

- Не коммить `config.json` с реальными ключами — добавь в `.gitignore`
- По каждому инструменту — не более одной позиции одновременно
- `rerun_interval_s` рекомендуется 60 секунд — данные тикера обновляются раз в секунду, частый опрос не нужен
