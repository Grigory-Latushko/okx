# margin06 — Pair Trading: BTC LONG + ETH SHORT

## Стратегия

Парная торговля на двух коррелированных инструментах.

Одновременно открываются **два ордера** на одинаковую сумму:
- **LONG** BTC-USDT-SWAP
- **SHORT** ETH-USDT-SWAP

Ставка на опережение BTC относительно ETH — бот зарабатывает когда BTC растёт сильнее ETH (или ETH падает сильнее BTC). Встроенный хедж снижает зависимость от общего направления рынка.

## Логика работы

```
Цикл:
  1. Проверяем позиции по BTC и ETH
  2. Если обе открыты → управляем трейлинг-стопами
  3. Если одна закрылась → ждём закрытия второй
  4. Если обе закрыты → открываем новую пару
```

## TP / SL

| Параметр | Значение |
|---|---|
| TP активация | 2 × ATR от цены входа |
| SL активация | 1 × ATR от цены входа |
| TP callback | `callback_atr_multiplier × ATR / цена` |
| SL callback | `callback_sl_atr_multiplier × ATR / цена` |

ATR считается **независимо** для BTC и ETH — каждый инструмент имеет свои TP/SL.
Ордера закрываются **независимо** — если ETH достиг TP, BTC продолжает жить со своим трейлингом.

## Конфиг (`config_60m.json`)

| Поле | Описание |
|---|---|
| `btc_instrument` | Инструмент для LONG (обычно `BTC-USDT-SWAP`) |
| `eth_instrument` | Инструмент для SHORT (обычно `ETH-USDT-SWAP`) |
| `position_size_usd` | Размер позиции в USD (одинаковый для обоих) |
| `leverage` | Плечо |
| `tp_atr_multiplier` | Множитель ATR для TP (рекомендуется `2.0`) |
| `sl_atr_multiplier` | Множитель ATR для SL (рекомендуется `1.0`) |
| `callback_atr_multiplier` | Callback для TP трейлинга |
| `callback_sl_atr_multiplier` | Callback для SL трейлинга |
| `dryRun` | `true` — тестовый режим без реальных сделок |

## Запуск

```powershell
# Тестовый режим (dryRun=true в конфиге)
.\trade.ps1

# Боевой режим
.\trade.ps1 -ForceLive

# Debug вывод
.\trade.ps1 -DebugMode
```

## ⚠️ Важные замечания

1. **Частичное открытие**: если BTC открылся, а ETH нет — бот логирует предупреждение. Закрой BTC вручную.
2. **Корреляция**: BTC и ETH не всегда движутся вместе. В периоды decoupling возможны убытки по обеим позициям одновременно.
3. **API ключи**: не коммить `config_60m.json` с реальными ключами — добавь в `.gitignore`.
